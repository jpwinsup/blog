winget download コマンドの FAQ

はじめに

この記事では、winget download を使用した Microsoft Store アプリのオフライン パッケージのダウンロードに関連して、これまで寄せられた問い合わせ (FAQ) に回答します。参考になれば幸いです。

よくある質問 (FAQ)

Q1. ダウンロードしたいオフラインパッケージの名前をどのように確認しますか？

A: winget search コマンドの結果から PackageID を確認することができます。

例えば、Windows 11 標準の電卓 (Windows Calculator) アプリのオフラインパッケージの PackageID を確認したい場合は、次のように検索できます。検索語句は部分一致したものがすべて列挙されるので、複数表示された場合は、その中から目的のアプリを見つけます。

※ 実際の検索結果には、複数のアプリが表示されます。
この場合、PackageID は 9WZDNCRFHVN5 ですので、次のように実行すれば、確実に Windows Calculator のオフラインパッケージを指定できます。

なお、PackageID を指定しなくても、winget download Calculator などと実行することもできますが、部分一致で検索するため、必ずしも目当てのアプリが選ばれるとは限りません。

Q2. オフラインパッケージはダウンロードされたが、Microsoft Store パッケージ ライセンスの取得でエラーが出る。

A: オフラインパッケージの入手と、ライセンスファイルの入手に必要な権限が異なるためにエラーが発生します。ユーザーに対して個別にオフラインパッケージでアプリをインストール・更新する用途であれば、オフラインパッケージの入手に成功していればライセンス取得でエラーが出ても問題ありません。
ライセンスファイルが必要なのは、一般的には Windows の標準搭載アプリ (プロビジョニング パッケージ) として構成する場合です。その場合、以下のいずれかのロールを持つ Microsoft Entra ID アカウントが必要です。

  • Global Administrator (全体管理者)
  • User Administrator (ユーザー管理者)
  • License Administrator (ライセンス管理者)

オフラインパッケージのみ取得する場合は、有効な Microsoft Entra ID アカウントがあれば特別な権限は不要です。
また、次のように –skip-license オプションで実行すれば、オフラインパッケージのみ入手して、ライセンスの取得をスキップすることもできます。

Q3. 特定の Microsoft Entra ID アカウントでダウンロードしたい

A: winget download のオプションに –authentication-account オプションを使用すれば、任意の Microsoft Entra ID アカウントを指定して実行可能です。

Q4. オフラインパッケージのダウンロード中にエラーで失敗する

A: お使いのネットワーク環境に依存して発生することが多く報告されています。現在使用しているネットワークの状態を確認し、可能であれば別のネットワークに切り替えてみてください。それでも解決しない場合は、Windows サポートチームに問い合わせることもご検討ください。また、お問い合わせの際は、次の手順で winget のログをあらかじめ取得いただけるとスムーズに進む可能性があります。

  1. winget download コマンドに verbose オプションをつけて実行して、オフラインパッケージのダウンロード中のエラーを再現します。

  2. 以下のコマンドを実行すると、winget のログのフォルダー (DiagOutputDir) が開きます。DiagOutputDir フォルダーごと回収して、サポート エンジニアにご連携ください。
