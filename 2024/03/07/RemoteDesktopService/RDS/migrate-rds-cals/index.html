RDS CAL のサーバー間移行手順について

Last Update: 2024-03-07
Remote Desktop►Remote Desktop Service (RDS)
feedback 共有

本記事はマイクロソフト社員によって公開されております。
こんにちは、Windows Commercial Support User Experience チームの佐々木です。
本投稿では、RDS CAL のサーバー間移行手順についてご案内させていただきます。

公開情報：
リモート デスクトップ サービス クライアント アクセス ラインセンス (RDS CAL) を移行する
https://learn.microsoft.com/ja-jp/windows-server/remote/remote-desktop-services/migrate-rds-cals

シナリオ例

以下の 3 つ移行のパターンに関してそれぞれ手順を説明します。

  1. 一台のライセンスサーバーに複数のライセンスパックをインストールしており、任意のライセンスパックを削除したい
  2. インストール済みのライセンスパックの割り当て数を変更したい
  3. ライセンスサーバー移行に伴い、ライセンスも移行したい

2. インストール済みのライセンスパックの割り当て数を変更したい

インストール済みのライセンスパックの割当数を変更いただくことはできません。
割当数を増やしたい場合は、ライセンスの追加インストール、
割当数を減らしたい場合は、一度ライセンスを削除し、再度任意の割当数でライセンスをインストールいただく必要がございます。

任意の割当数でライセンスをインストールいただく際は、後述の電話の欄に記載があるクリアリングハウスに連絡する方法で、ライセンス数を任意の割当数で発行いただく必要があります。

また、ライセンスを削除いただいても、すでにライセンスの付与されたユーザーもしくはデバイスの場合、 接続有効期限内であれば、接続不可となることはありません。

ただし、ライセンスの付与されていないユーザーもしくはデバイスからの新規接続、および
有効期限切れのユーザーもしくはデバイスからの接続は不可となります。

3. ライセンスサーバー移行に伴い、ライセンスも移行したい

以下の公開情報の補足情報を記載します。

公開情報：
リモート デスクトップ サービス クライアント アクセス ラインセンス (RDS CAL) を移行する
https://learn.microsoft.com/ja-jp/windows-server/remote/remote-desktop-services/migrate-rds-cals

ライセンスの移行は、購入証書等をご用意の上、下記 3 通りの方法で実施いただけます。
・自動接続
・Web ブラウザー
・電話

自動接続、Web ブラウザー

1.サーバーマネージャー -> ツール -> RemoteDesktopServices -> リモートデスクトップライセンスマネージャー -> すべてのサーバー -> 移行先のライセンスサーバー 。
2.右クリック -> ライセンスの管理 を開きます。
3.表示されるダイアログに沿って移行を実施ください。

移行実施時に、移行元と移行先が疎通可能なな場合には自動的に移行元よりライセンスが削除されますが、
疎通が取れない場合には移行元にもライセンスがインストールされたままとなりますため、
手動でライセンスの削除および RD ライセンス サーバーの機能を削除します。

電話

1.前述のページ内 [RDS CAL の移行方法を確立する] 項に記載の手順を
移行先のライセンス サーバーにて実施します。
※ [接続方法] は [電話] を選択してください
※ [クライアント ライセンス キー パックの取得] ページの表示まで実施

2.下記にお電話をいただき、オペレーターに、
ライセンス サーバーの新規構築に伴い、現行のライセンス サーバーにて管理している
RDS (TS) CAL を移行されたい旨をお伝えください。

購入証書を参照の上、以下の情報をクリアリングハウスのオペレーターにお伝えください。

  1. 法人名 (ライセンスの保有者)
  2. 担当者漢字フルネーム
  3. ライセンス サーバーID
  4. ボリューム ライセンスのプログラム名
  5. 契約番号
  6. RDS CALのバージョン
  7. RDS 環境の OS バージョン
  8. CALの種類と数

3.オペレーターよりライセンス キー パック ID をお伝えいたします。
4.移行先のライセンス サーバーにて、ライセンス キー パック ID を入力してください。
5.移行先のライセンス サーバーにて、対象の RDS (TS) CAL が移行されていることを確認してください。
6.移行元のライセンス サーバーにて、対象の RDS (TS) CAL を手動で削除してください。

よくある質問

Q. ライセンス移行後に、移行元のサーバーからライセンスが発行されたユーザーもしくはデバイスは、接続不可となるか。

A. いいえ、すでにライセンスの付与されたユーザーもしくはデバイスの場合、
　接続有効期限内であれば、接続不可となることはありません。
ただし、ライセンスの付与されていないユーザーもしくはデバイスからの新規接続、および有効期限切れのユーザーもしくはデバイスからの接続は不可となります。
