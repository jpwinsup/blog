2020 年 2 月 25 日以降の累積更新プログラムを適用すると、NCSI によるインターネット接続性の確認が "インターネット なし" と判定されることがある。

Last Update: 2021-06-25

こんにちは。Windows プラットフォーム サポートです。

Windows 10 Version 1709, 1803, 1809, 1903, 1909 および Windows Server 2019 で 2020 年 2 月 25 日以降 3 月 25 日までにリリースされた累積更新プログラムを適用すると、後述の 2 つの問題が発生することが判明しましたので、問題の概要と解決方法について本 Blog でご案内します。

[ 問題 1 の概要 ]
タスクトレイのネットワークアイコンの状態が 「インターネット アクセスなし」 となる場合があります。

または

これにより、Microsoft Office, Office365, Outlook などのアプリケーションの動作に問題が発生する場合があります。

[ 問題 2 の概要 ]
VPN を切断後、ユーザーはインターネット上のサイトにアクセスできなくなり、プロキシにアクセスできないことを示すエラーページが表示される場合があります。

[ 対象 OS ]
Windows 10 Version 1709
Windows 10 Version 1803
Windows 10 Version 1809 / Windows Server 2019
Windows 10 Version 1903
Windows 10 Version 1909

[ 原因 ]
該当の更新プログラムに含まれる winhttp.dll の修正の影響によって問題が発生します。

[ 発生条件 ]
以下の更新プログラムを適用した場合に発生する可能性があります。

[ 対応策 ]
米国時間 2020/3/30 (日本時間 3/31) 付けで本問題を修正する更新プログラムを定例外で公開しました。

Windows 10 / Windows Server の各バージョンに対応した更新プログラムを以下の通りリリースしておりますので、ご利用環境に即したものを適用ください。

Windows 10, version 1709 KB4554342
Windows 10, version 1803 KB4554349
Windows 10, version 1809 / Windows Server 2019 KB4554354
Windows 10, version 1903 KB4554364
Windows 10, version 1909 KB4554364

なお、上記の更新プログラムは、現時点では弊社 Microsoft Update Catalog からの配布のみとなっております。
WSUS をご利用の場合は、Catalog から更新プログラムをインポートすることで、管理している端末に更新プログラムを展開することができます。
更新のインポート手順については下記サイトをご参照ください。

WSUS and the Catalog Site

また、Windows 10 / Windows Server 2019 の更新プログラムはすべて累積的であり、日本時間 4 月 15 日に公開の 4 月の月例累積更新プログラムにも同様の修正が含まれております。今後、公開予定の累積更新プログラムでも修正済みです。
