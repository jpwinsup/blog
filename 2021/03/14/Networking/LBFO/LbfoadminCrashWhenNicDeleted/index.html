メンバー NIC が故障すると、チーミングのプロパティが表示されなくなる

Last Update: 2021-03-15
Networking►LBFO
feedback 共有

概要

NIC チーミング (LBFO) のメンバー NIC が故障した場合、以下の手順で NIC チーミングのプロパティが開かなくなる事象が発生します。

  1. サーバー マネージャーを開きます。
  2. [ローカル サーバー] をクリックし、[NIC チーミング] - [有効] をクリックします。
  3. 左下の [チーム] の項目から、チーム名を右クリックし、[プロパティ] をクリックします。
  4. プロパティ画面が開かないことが確認できます。

また、アプリケーション イベントログには lbfoadmin.exe がクラッシュした旨のイベントが記録されます。

現象発生 OS

  • Windows Server 2022
  • Windows Server 2019
  • Windows Server 2016
  • Windows Server 2012/2012R2

原因

メンバー NIC の情報を正しく取得できなくなることが原因で、問題が発生します。
この問題が発生する主なシナリオは、メンバー NIC が突然故障した場合や、メンバー NIC が取り外された場合が挙げられます。

影響

本問題により、LBFO のプロパティ画面が表示されず、GUI から設定の確認や変更ができない影響が発生します。
ただし、ネットワーク通信は存在しているメンバー NIC で行われているため、通信そのものに影響はありません。

回避策

現象が発生した場合には、対象の NIC をチームメンバーから削除するため、以下の PowerShell コマンドを管理者権限で実行してください。

$RemovedNic = Get-NetLbfoTeamMember | Where-Object {$_.FailureReason -eq “NicNotPresent”}
Remove-NetLbfoTeamMember -Name $RemovedNic.Name -Team $RemovedNic.Team -Confirm:$false

参考情報

Get-NetLbfoTeamMember
https://docs.microsoft.com/en-us/powershell/module/netlbfo/get-netlbfoteammember?view=win10-ps

Remove-NetLbfoTeamMember
https://docs.microsoft.com/en-us/powershell/module/netlbfo/remove-netlbfoteammember?view=win10-ps
