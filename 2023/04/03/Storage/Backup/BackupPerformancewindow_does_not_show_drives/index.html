バックアップ パフォーマンスの設定に一部のボリュームが表示されない事象について

Last Update: 2023-04-03
Storage►Backup
feedback 共有

本記事は、マイクロソフト社員によって公開されております。

こんにちは、Windows サポートチームの三浦です。
本日は Windows Server バックアップのバックアップ パフォーマンスの設定画面に一部のボリュームが表示されない事象について紹介させていただきます。
バックアップ パフォーマンス設定の概要については、こちら の記事をご参照ください。

事象概要

Windows Server バックアップでは、完全/増分バックアップのどちらの方式のバックアップを実行するかを設定することが可能です。
この設定は [バックアップ パフォーマンスの最適化] 画面から指定が可能であり、カスタム設定を行うことで、ボリュームごとに完全/増分バックアップの指定が可能です。

しかしながら、このカスタム欄に表示されるボリューム一覧に一部のボリュームが表示されない事象が発生する場合があることが確認できております。
下記は事象の発生例となり、F ドライブが存在するにもかかわらずカスタム欄のボリューム一覧に表示されておりません。


発生条件

Windows Server 2016 以降の UEFI ベースの Serve OS で発生する可能性がございます。
(BIOS 環境においても、Read Only あるいは FAT などの非 NTFS ボリュームが存在する環境の場合は発生する可能性がございます。)

発生原因

本問題は現在弊社で調査中の状況であり、現時点では、カスタム欄に表示するボリュームの選定処理の問題に起因して発生する可能性があることが確認できております。
将来の修正に向けて対応中ではございますが、本問題の修正のリリースは現段階では未定となりますので、ボリュームごとに完全/増分バックアップの指定を行いたい場合は、後述の回避策にて設定を行うことをご検討ください。

なお、本問題による影響は、[バックアップ パフォーマンスの最適化] 画面の表記上の問題に留まり、実際の完全/増分バックアップの実行自体には影響いたしません。

回避策

PowerShell コマンドを利用することで、本問題の影響を回避して完全/増分バックアップの確認や指定が可能でございます。
以下に手順例をご案内いたしますので、ボリュームごとに完全/増分バックアップの指定が必要な場合は、本回避策をご利用ください。

  1. 管理者権限の PowerShell を起動します。

  2. 下記コマンドを実行し、設定対象のボリューム情報を取得して変数に格納します。

// 実行例
$Vol=Get-WBVolume -VolumePath “E:”

  1. 続いて下記コマンドを実行し、パフォーマンス設定を変更します。

// 完全バックアップに設定する場合
Set-WBPerformanceConfiguration -Volume $Vol AlwaysFulll

// 増分バックアップに設定する場合
Set-WBPerformanceConfiguration -Volume $Vol AlwaysIncremental

  1. 設定値を確認したい場合は、Get-WBPerformanceConfiguration コマンドを実行ください。
  • 参考情報
    Set-WBPerformanceConfiguration
    Get-WBPerformanceConfiguration
